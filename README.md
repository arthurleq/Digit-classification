# Digit Recognizer

A handwritten digit recognition application built with a CNN (Convolutional Neural Network) trained on MNIST, with an interactive C++ interface using SFML.

---

## Demo

Launch digit_recognizer.exe, draw a digit in the window, and the model predicts the **top 3 most likely digits** with confidence percentages in real time.

---

## Project Structure

```
Digit-classification/
├── main.cpp              # C++ application (SFML + TensorFlow C API)
├── CMakeLists.txt        # CMake build configuration
├── training.ipynb        # CNN training notebook (Keras/TensorFlow)
├── convert_model.py      # Converts model.h5 → SavedModel format
├── model.h5              # Trained Keras model
├── saved_model/          # TensorFlow SavedModel (generated by convert_model.py)
├── requirements.txt      # Python dependencies
└── README.md
```

## Dependencies

### C++ (Runtime)
- [SFML](https://www.sfml-dev.org/) >= 2.5 — graphics, window, system
- [TensorFlow C API](https://www.tensorflow.org/install/lang_c) — model inference

### Python (Training)
- Python 3.x
- TensorFlow / Keras
- See `requirements.txt` for the full list

```bash
pip install -r requirements.txt
```

## Getting Started

### 1. Train the model (optional)

Open and run `training.ipynb` in Jupyter to train the CNN on MNIST. This produces `model.h5`.

### 2. Convert the model

```bash
python convert_model.py
```

This converts `model.h5` into the `saved_model/` directory (TensorFlow SavedModel format), required by the C++ application.

### 3. Build the C++ application

```bash
mkdir build && cd build
cmake ..
make
```

### 4. Run

```bash
./digit_recognizer
```

---

## Usage

- **Draw** a digit (0–9) on the white canvas with your mouse
- **Release** the mouse button to trigger the prediction
- The right panel displays the **top 3 predictions** with confidence bars
- **Click again** to clear the canvas and draw a new digit

---

## Model Architecture

The CNN is trained on the [MNIST dataset](https://en.wikipedia.org/wiki/MNIST_database) and exported using TensorFlow's SavedModel format for inference via the TensorFlow C API.

Input: 28×28 grayscale image (the canvas is downscaled from 280×280 via average pooling)
